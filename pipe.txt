Pipeline end-to-end (không thêm LLM)
0) Offline indexing (làm trước, không nằm trong infer latency)

Input: bài báo thô (HTML/PDF text)
Process:

Clean + normalize text (remove boilerplate, nav, ads).

Chunking theo đoạn:

chunk_tokens = 200~300

overlap = 40~80

Với mỗi chunk, lưu:

chunk_id, doc_id, url, publisher, title, published_at, chunk_text

Tính embedding chunk bằng bi-encoder (vd: bge-small/e5-small):

chunk_emb = embed(chunk_text)

Index:

Vector index (FAISS/HNSW) trên chunk_emb

(optional) BM25 index trên chunk_text để hybrid

Output: VectorIndex + MetadataStore (+ BM25)

Inference pipeline (thời gian thực)
Step 1) Preprocess claim

Input: claim_text
Process:

normalize (lowercase, strip, unify numbers/date format nếu cần)

(optional) extract entities (NER/rules) để tạo filters

Output: claim_norm

Step 2) Bi-encoder retrieve 50 chunks (fast recall)

Input: claim_norm
Process:

q_emb = embed(claim_norm)

Vector search top 50:

candidates = ANN.search(q_emb, topK=50)

(optional) Hybrid merge với BM25 top 50 rồi union → cỡ 80–100, sau đó cắt về 50 theo score tổng.

Output (list 50):

[
  { "chunk_id": "...", "doc_id": "...", "text": "...", "meta": {...}, "bi_score": 0.73 },
  ...
]

Step 3) Cross-encoder rerank (precision, vẫn nhanh)

Input: 50 candidates + claim
Process:

chạy cross-encoder trên cặp (claim, chunk_text) để ra ce_score

ví dụ model: bge-reranker-small / MiniLM cross-encoder

sort giảm dần theo ce_score

Output: same list nhưng có ce_score

Step 4) Select top 8 chunks (token budget control)

Input: ranked candidates
Process:

lấy top_chunks = top 8

(optional) “publisher gating”:

max 1 chunk / domain nếu bạn muốn đa dạng nguồn

nếu trùng domain nhiều, thay bằng chunk tiếp theo

Output (<=8):

[
  { "chunk_id": "...", "doc_id": "...", "text": "...", "ce_score": 12.3, "meta": {...} },
  ...
]

Step 5) Sentence select top 2 mỗi chunk (extractive, không LLM)

Input: top_chunks + claim_norm
Process cho từng chunk:

Split chunk thành câu (sentence splitter: spaCy, underthesea…)

Score từng câu bằng cách nhẹ:

Option A (khuyến nghị): embedding sentence (precompute offline cũng được)

s_score = cosine(embed(claim), embed(sentence))

Option B: BM25/TF-IDF overlap

Option C: mix A+B

Chọn top 2 sentences theo s_score

Output:

[
  {
    "chunk_id": "...",
    "sent_top": [
      { "sid": 5, "sent": "...", "s_score": 0.81 },
      { "sid": 2, "sent": "...", "s_score": 0.78 }
    ],
    "all_sents": [...]
  },
  ...
]

Step 6) Window ±1 câu quanh mỗi câu top (giữ ngữ cảnh tối thiểu)

Input: kết quả Step 5
Process:

Với mỗi câu top (sid):

lấy sent[sid-1], sent[sid], sent[sid+1] nếu tồn tại

Merge window của 2 câu top trong cùng chunk (tránh lặp)

Output thành 1 snippet ngắn cho mỗi chunk

Output (snippets):

[
  {
    "doc_id": "...",
    "chunk_id": "...",
    "snippet": "Sentence 4... Sentence 5... Sentence 6... Sentence 1... Sentence 2... Sentence 3...",
    "meta": { "url": "...", "publisher": "...", "published_at": "..." },
    "scores": { "ce_score": 12.3 }
  },
  ...
]

Step 7) Dedup snippets (tránh 5 nguồn nói cùng 1 ý)

Input: list snippets
Process (nhanh):

Tính fingerprint (SimHash/MinHash) hoặc embedding snippet

Nếu similarity > threshold (vd 0.90) thì bỏ bản trùng (ưu tiên giữ snippet có ce_score cao hơn / nguồn uy tín hơn)

Output: unique_snippets (thường còn 4–8)

Step 8) Build final LLM context (only once)

Input: claim + unique_snippets
Process:

Format “Evidence Cards” (không tóm tắt):

Source, date, url, snippet (ngắn)

Enforce token budget:

nếu vượt, cắt bớt snippet thấp điểm nhất trước

LLM Input mẫu:

System: hướng dẫn fact-check + cite sources

User:

Claim: ...

Evidence:

[Publisher | Date | URL] snippet…

...

Output: verdict + explanation + citations (theo url/publisher)

Tham số khuyến nghị để chạy nhanh

Chunk: 200–300 tokens, overlap 50

Retrieve: topK=50

Rerank: cross-encoder trên 50 (batch inference)

Top chunks: 8

Sentence select: 2 câu/chunk

Window: ±1 câu

Dedup threshold: 0.90 (embedding) hoặc simhash distance ngưỡng phù hợp

Evidence budget: 800–1500 tokens (tuỳ context LLM)
